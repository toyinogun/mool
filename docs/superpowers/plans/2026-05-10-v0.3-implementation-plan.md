# Mool v0.3 — Webcam Picture-in-Picture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a camera-on toggle to the Recorder page (default off). When on, a live mirrored preview circle appears below the toggle row; the camera stream is held by the adapter from toggle-on until toggle-off. When the user clicks Start, a hidden canvas at the screen's native resolution composites the screen frame and a circle-clipped, un-mirrored webcam in the bottom-left at 18% of canvas height with a 4% margin. `canvas.captureStream(30)` produces the recorded video track; mic audio is merged in by `recorderCapture.js` exactly as in v0.2.

**Architecture:** Extend the existing pure-reducer + adapter split. The reducer (`recorderFlow.js`) widens four payload shapes to carry a `videoEnabled: boolean` flag (`StartClicked`, `RequestingMic`, `Starting`, `startRecording`); no new state variants, transitions, or effects. The camera stream lives entirely in the adapter (`recorder.js`), not the reducer — its lifecycle is driven by the toggle, not by Start, so there is no recording-flow state to model. A new `recorderComposite.js` module encapsulates the canvas + rAF compositing and exposes `composeStreams({ screenStream, cameraStream })` returning `{ compositeStream, stop, onCameraEnded }`. Audio merging stays in `recorderCapture.js` as a single source of truth (see Task 5 note). Server, R2 layout, and SQLite schema are unchanged.

**Tech Stack:** Node.js + Express + TypeScript, plain HTML + vanilla JS (ES modules, no bundler), `MediaRecorder` + `getUserMedia` + `getDisplayMedia` browser APIs, `<canvas>` + `requestAnimationFrame` + `canvas.captureStream`, vitest + supertest for tests.

**Spec:** `docs/superpowers/specs/2026-05-10-v0.3-webcam-pip-design.md`

## Task ordering rationale

Each task leaves a working tree (`vitest run` green, `tsc --noEmit` clean, no half-implementations the next task depends on):

1. **Reducer threading first** (Task 1). Pure functions, fully unit-tested. The flag is plumbed through reducer state and effect payload before anything reads it; `videoEnabled: false` is the only value the production path ever produces until the camera toggle exists.
2. **Effects dispatcher** (Task 2). One-line update to forward the new effect field. Pinned by a `recorderEffects.test.ts` update.
3. **Static UI** (Task 3). HTML + CSS for the toggle, preview circle, and hide-link. Visible but inert — no behavior wired yet.
4. **Composite module** (Task 4). New file, freestanding. No integration yet; smoke-testable in isolation if desired but no unit tests (canvas/rAF are DOM-coupled).
5. **Adapter — camera lifecycle** (Task 5). Toggle on requests camera, renders preview, holds the stream; toggle off releases it. Hide/Show preview is the same DOM file. With this task the preview works end-to-end on the page even though Start with camera-on doesn't yet composite (the toggle ref is read but the held stream is ignored — `videoEnabled` always reaches the port as `true` if the user toggles cam on, but the port stays on the v0.2 path until Task 6).
6. **Adapter — composite wiring** (Task 6). The `startCapture` port branches: `videoEnabled` true and a held camera stream → composite then record; otherwise v0.2 path. Cleanup is wired into `releaseStream`.
7. **Manual smoke matrix** (Task 7). Eight scenarios from §10 of the spec.
8. **Final checks + PR** (Task 8). Type check, full suite, Docker build, PR.

## Task 4: New module — `recorderComposite.js`

**Goal:** Create the canvas + `requestAnimationFrame` compositor. Exposes `composeStreams({ screenStream, cameraStream }) → { compositeStream, stop, onCameraEnded }`. Composite is video-only; audio merging stays in `recorderCapture.js`. The screen track ending naturally propagates as the composite track ending so the adapter's `TrackEnded` wiring catches it.

**Files:**
- Create: `app/src/public/recorderComposite.js`

This module is DOM-coupled (canvas, off-screen video elements, rAF). Per the spec §10 it gets no unit tests; correctness is verified in Task 7 (manual smoke). The module's surface is small and its logic is "draw screen, then maybe draw cam in circle."

- [ ] **Step 1: Create the file**

Create `app/src/public/recorderComposite.js`:

```javascript
/**
 * Canvas-based picture-in-picture compositor for the Recorder page.
 *
 * Takes a screen MediaStream and a camera MediaStream and returns a single
 * composite MediaStream whose video track is a hidden <canvas> drawn by a
 * requestAnimationFrame loop: the screen frame as the base layer, then the
 * webcam clipped to a circle in the bottom-left at 18% of canvas height
 * with a 4% margin from both edges. The webcam frame is centre-cropped to
 * a square before draw so 16:9 or 4:3 webcam feeds aren't distorted by the
 * circular clip.
 *
 * The composite is video-only. Audio merging stays in recorderCapture.js
 * (single source of truth for mime-type selection); the adapter passes
 * audioStream straight to capture.start() as in v0.2.
 *
 * The screen track ending (e.g. user clicks the browser's "Stop sharing"
 * toolbar) is propagated as the composite video track ending, so
 * recorderCapture.js's onTrackEnded wiring catches it as TrackEnded →
 * Stopping. The camera track ending (e.g. USB cam unplugged) is propagated
 * via the onCameraEnded callback only — the composite continues drawing
 * screen-only, MediaRecorder is unaffected, and the recording gracefully
 * becomes screen-only from that point.
 *
 * @typedef {{
 *   compositeStream: MediaStream,
 *   stop: () => void,
 *   onCameraEnded: (cb: () => void) => void,
 * }} Composite
 */

/**
 * @param {{ screenStream: MediaStream, cameraStream: MediaStream }} args
 * @returns {Composite}
 */
export function composeStreams({ screenStream, cameraStream }) {
  const screenTrack = screenStream.getVideoTracks()[0];
  const cameraTrack = cameraStream.getVideoTracks()[0];
  if (!screenTrack) throw new Error('composeStreams: screenStream has no video track');
  if (!cameraTrack) throw new Error('composeStreams: cameraStream has no video track');

  const settings = screenTrack.getSettings();
  const canvasWidth = settings.width || 1920;
  const canvasHeight = settings.height || 1080;

  const canvas = document.createElement('canvas');
  canvas.width = canvasWidth;
  canvas.height = canvasHeight;

  const ctx = canvas.getContext('2d');
  if (!ctx) throw new Error('composeStreams: 2d canvas context unavailable');

  // Off-screen video elements feed the canvas. Both are muted (audio never
  // flows through here) and play immediately so currentTime/frames are live.
  const screenVideo = document.createElement('video');
  screenVideo.muted = true;
  screenVideo.autoplay = true;
  screenVideo.playsInline = true;
  screenVideo.srcObject = screenStream;
  void screenVideo.play().catch(() => {
    /* play() rejects if the element is removed before it resolves; harmless. */
  });

  const cameraVideo = document.createElement('video');
  cameraVideo.muted = true;
  cameraVideo.autoplay = true;
  cameraVideo.playsInline = true;
  cameraVideo.srcObject = cameraStream;
  void cameraVideo.play().catch(() => {});

  // PIP geometry per spec §7.
  const pipDiameter = canvasHeight * 0.18;
  const pipRadius = pipDiameter / 2;
  const pipMarginX = canvasWidth * 0.04;
  const pipMarginY = canvasHeight * 0.04;
  const pipCx = pipMarginX + pipRadius;
  const pipCy = canvasHeight - pipMarginY - pipRadius;

  let cameraActive = true;
  /** @type {Array<() => void>} */
  const cameraEndedCallbacks = [];

  cameraTrack.addEventListener('ended', () => {
    cameraActive = false;
    for (const cb of cameraEndedCallbacks) {
      try { cb(); } catch { /* swallow — one bad listener shouldn't stop others */ }
    }
  });

  /** @type {number | null} */
  let rafId = null;
  let stopped = false;

  function draw() {
    if (stopped) return;
    rafId = requestAnimationFrame(draw);

    // Base: screen frame.
    if (screenVideo.readyState >= 2) {
      try {
        ctx.drawImage(screenVideo, 0, 0, canvasWidth, canvasHeight);
      } catch {
        /* drawImage can throw if the element is detached mid-frame; ignore. */
      }
    }

    // Overlay: webcam, centre-cropped to a square then drawn into a circular clip.
    if (cameraActive && cameraVideo.readyState >= 2) {
      const camW = cameraVideo.videoWidth;
      const camH = cameraVideo.videoHeight;
      if (camW > 0 && camH > 0) {
        // Centre-crop source to square.
        const side = Math.min(camW, camH);
        const sx = (camW - side) / 2;
        const sy = (camH - side) / 2;

        ctx.save();
        ctx.beginPath();
        ctx.arc(pipCx, pipCy, pipRadius, 0, Math.PI * 2);
        ctx.closePath();
        ctx.clip();
        try {
          ctx.drawImage(
            cameraVideo,
            sx, sy, side, side,
            pipCx - pipRadius, pipCy - pipRadius, pipDiameter, pipDiameter,
          );
        } catch {
          /* same defensive ignore as above */
        }
        ctx.restore();
      }
    }
  }

  rafId = requestAnimationFrame(draw);

  // The composite's video track. captureStream samples the canvas at 30 fps
  // regardless of how often draw() runs.
  /** @type {any} */
  const captureStreamFn =
    /** @type {any} */ (canvas).captureStream
    || /** @type {any} */ (canvas).mozCaptureStream;
  if (!captureStreamFn) {
    throw new Error('composeStreams: canvas.captureStream is not supported in this browser');
  }
  const compositeStream = /** @type {MediaStream} */ (captureStreamFn.call(canvas, 30));

  // When the screen track ends (user clicks the browser's Stop-Sharing toolbar),
  // stop the composite video track so capture's onTrackEnded wiring fires.
  screenTrack.addEventListener('ended', () => {
    for (const t of compositeStream.getVideoTracks()) {
      try { t.stop(); } catch { /* idempotent */ }
    }
  });

  function stop() {
    if (stopped) return;
    stopped = true;
    if (rafId !== null) {
      cancelAnimationFrame(rafId);
      rafId = null;
    }
    for (const t of compositeStream.getVideoTracks()) {
      try { t.stop(); } catch { /* idempotent */ }
    }
    // Release the off-screen video elements' refs so the GC can reclaim them
    // and the underlying source-stream readers wind down.
    screenVideo.srcObject = null;
    cameraVideo.srcObject = null;
  }

  return {
    compositeStream,
    stop,
    onCameraEnded(cb) {
      cameraEndedCallbacks.push(cb);
    },
  };
}
```

## Notes for the implementer

- **Audio merging stays in `recorderCapture.js`.** The spec §7 says `composeStreams` `addTrack`-s the audio. The plan diverges slightly: composite is video-only and `recorderCapture.js` handles audio merging exactly as in v0.2. Reason: keeping mime-type selection (`pickMimeType(Boolean(audioStream))`) and audio integration in one module avoids duplication and an easy-to-miss "composite has audio but capture chose a no-audio mime type" bug.
- **The reducer never references `cameraStream`.** Camera lives in the adapter; the reducer only carries `videoEnabled` forward. If you find yourself adding camera-stream references to `recorderFlow.js`, stop and re-read spec §6.
- **Manual smoke is load-bearing.** v0.1 set the line of "no jsdom, no Playwright" deliberately; v0.3 holds it. The composite module's correctness is verified by Task 7, not by unit tests. If a smoke fails, the bug is real — do not paper over it with a unit test that mocks the canvas.
- **Permissions persistence varies by browser.** Chrome remembers a "Block" decision per origin; Safari and Firefox prompt more often. If a smoke step's permission prompt doesn't appear, that's usually a stale browser-side decision, not a bug — reset the site permission and re-run.
